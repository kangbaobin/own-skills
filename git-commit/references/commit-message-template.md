# Commit Message 模板参考

## 格式

```
<type>(<scope>): <subject>

<body>
```

## 字段说明

### Type（必填）

| Type       | 使用场景                               | 举例                           |
|------------|----------------------------------------|-------------------------------|
| `feat`     | 新增功能、接口、页面                   | 新增用户注册接口               |
| `fix`      | 修复已知 bug                           | 修复登录态丢失问题             |
| `docs`     | README、注释、API 文档                 | 补充配置说明文档               |
| `style`    | gofmt、缩进、空行、命名（不改逻辑）   | 统一代码缩进风格               |
| `refactor` | 重构但不改变外部行为                   | 提取公共方法到 utils 包        |
| `perf`     | 性能优化                               | 优化批量查询的 N+1 问题        |
| `test`     | 新增或修改测试用例                     | 补充订单服务的单元测试         |
| `chore`    | CI/CD、Makefile、依赖管理等            | 升级 Go 版本至 1.22            |

### Scope（可选）

简要说明影响的模块、包或文件。

```
feat(auth):        → 影响 auth 模块
fix(handler/user): → 影响 handler/user 包
chore(Makefile):   → 影响 Makefile
```

### Subject（必填）

- 中文撰写
- ≤ 50 字符
- 祈使语气（"添加"而不是"添加了"）
- 不加结尾句号
- 文件重命名/移动须标注路径

### Body（必填）

- 中文撰写
- 每行 ≤ 72 字符
- 回答两个问题：
  1. **为什么**要做这个改动？（动机、背景）
  2. **改了什么**？（具体变更内容）
- 破坏性改动加 `BREAKING CHANGE:` 段落

## 完整示例

```
feat(middleware): 添加请求限流中间件

线上高峰时段接口偶现超时，排查发现部分接口
缺少限流保护。基于令牌桶算法实现限流中间件，
支持按 IP 和用户维度分别配置 QPS 上限。
默认全局限制 1000 QPS，可通过配置文件覆盖。
```

```
fix(dao): 修复软删除记录被重复查询的问题

用户列表接口返回已注销用户，原因是 ListUsers
查询未加 deleted_at IS NULL 条件。在 BaseDAO
的通用查询方法中补充软删除过滤，同时为
已有的 28 个查询方法补充测试用例。
```

```
refactor(config): 统一配置加载为 Viper 方案

mv internal/config/env.go -> internal/config/viper.go

原有配置分散在环境变量和 JSON 文件中，维护成本
高且容易遗漏。迁移至 Viper 统一管理，支持
环境变量、配置文件、命令行参数三种来源，
按优先级自动合并。
```
